  Djang开发的基础包括了模型（Model）的设计，URL的配置，View（视图）的编写，Template（模板）的设计和Form(表单)的使用，今天就来总结一下model设计和注意事项。

什么是Model模型？

  Model (模型) 简而言之即数据模型。模型不是数据本身（比如数据库里的数据），而是抽象的描述数据的构成和逻辑关系。每个Django model实际上是个类，继承了models.Model。每个Model应该包括属性，关系（比如单对单，单对多和多对多）和方法。当你定义好Model模型后，Django的接口会自动帮你在数据库生成相应的数据表(table)。这样你就不用自己用SQL语言创建表格或在数据库里操作创建表格了，用起来很省心。

  • 这个表的名称Publisher和Book，是根据模型中的某些元数据自动生成的，也可以重写为别的名称。
  • id 字段是自动添加的，但这个行为可以被重写。
  • 这个例子中的CREATE TABLE SQL 语句使用PostgreSQL 语法格式，要注意的是Django 会根据settings文件 中指定的数据库类型来使用相应的SQL 语句。

模型的字段类型

  django自带的字段类型有10多种，具体见django官方文档

模型的字段选项

  每个字段类型都可以通过字段选项进行设置，具体的字段选项可以见django官方文档

模型设计注意事项

  • 这个Field是否有必选项, 比如CharField的max_length和ForeignKey的on_delete选项(django2.0)是必须要设置的。
  • 这个Field是否必需(blank = True or False)，是否可以为空 (null = True or False)。这关系到数据的完整性。
    • null 是针对数据库而言，如果 null=True, 表示数据库的该字段可以为空。
    • blank 是针对表单的，如果 blank=True，表示你的表单填写该字段的时候可以不填。
    • Django对于空白的CharField和TextField永远不会存为null空值，而是存储空白字符串’’，所以正确的做法是设置default=’’。
